Content Based Image Retrieval System

About this project

This project was a group school assignment completed in May 2022.

The objective of this project was to design and create a Content Based Image Retrieval system by applying both image processing and machine learning techniques.

Project outcomes

First, my teammates and I started the project by carrying out research on related topics. We looked into the methodologies or algorithms proposed by the other researchers to solve the same or similar problem.

After that, we derived and implemented a feasible solution for Content Based Image Retrieval (CBIR), which is a Support Vector Machine (SVM) that classifies the images using the following 3 types of features:

  • Color Moments (CM), which measures the mean and the standard deviation of each RGB colour component.
  • Local Binary Pattern (LBP), which represents the image texture. It was computed using the MATLAB built-in method called "extractLBPFeatures".
  • Histogram of Oriented Gradients (HOG), which represents the shapes of the objects within an image. It was computed using the MATLAB built-in method called "extractHOGFeatures".

Finally, we evaluated the strengths and weaknesses of our Content Based Image Retrieval (CBIR) system.

  • Strengths:
    1. It accepts RGB images in any size.
    2. It accepts unannotated images, which are images without any metadata such as captions and keywords.
  • Weaknesses:
    1. It does not accept grayscale images due to the usage of the Color Moments (CM) feature.
    2. It is not invariant to image rotation due to the usage of the Local Binary Pattern (LBP) and Histogram of Oriented Gradients (HOG) features.

The dataset used for model training and testing was adapted from the Intel Image Classification Dataset . It contains a total of 2800 RGB images distributed in the following 4 categories:

  • Building
  • Forest
  • Mountain
  • Sea
